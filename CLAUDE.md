# Boostly Marketing Site

## Goals
- Convert independent restaurant owners into Boostly trials
- Remove skepticism before they hit pricing — show the math, address objections
- Make the ROI undeniable: every section should move a skeptical owner closer to "this pays for itself"

## Tech Stack
- Plain HTML, CSS, and vanilla JavaScript — no frameworks, no build tools
- One file per page: `index.html` (landing page), `calculator.html` (ROI calculator)
- Styles are inline `<style>` blocks within each HTML file
- No npm, no bundler, no TypeScript

## Local Development
- Always use **port 4000** for the dev server in this project
- Preview at **http://localhost:4000**

## Copy Rules
- **Always reference `persona.md` when writing or editing any copy**
- Write for the restaurant owner persona: plain language, specific numbers, no jargon
- Lead with the pain, follow with the proof, close with the math
- Use their language: "guests" not "customers", "covers" not "users", "slow nights" not "low traffic periods"
- Every claim should feel like it came from a real owner, not a marketing team
